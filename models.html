Deep Learning Networks Used


1. Custom Convolution Neural Network (CNN)

Firstly, we created a custom CNN Network using Keras Framework. Code representing structure of the model can be seen below:


For the custom network 3 Conv2d layers with 8, 16 and 32 filters were connected end by end. All the filters had kernel size of 3, 3 and stride of 1. Relu activation unit was used on all layers. After the first layer Batch Normalization was performed followed by Max Pooling with kernel of 2,2. For the remaining two layers Batch Normalization was followed with Average Pooling of the same kernel size. The output was then flattened and passed into Dense layer with 150 filters and Tanh activation units. Finally, with the dropout rate of 0.5, they were all Normalized again and passed into SoftMax Unit having filters equal to number of class labels. Learning Rate was set to 0.0001 and with the batch size of 32 the model was trained for 14 epochs with Callback monitoring accuracy of validation set.


2. Pre-Trained CNN Networks

We also implemented 5 different pre-trained networks from Keras framework namely -

i. VGG16
ii. VGG19
iii. ResNet152V2
iv. InceptionV3
v. InceptionResV2

A code snippet of implementation of VGG19 model can be seen below:

The weights of pre-trained networks were fixed. The output from pretrained networks were flattened and passed to Dense layer with 150 filters and Tanh activation unit. Then with dropout rate of 0.5 the outputs were normalized and passed into a final Dense layer with 12 filters and SoftMax as activation unit.Finally, learning rate was set to 0.0001 and with the batch size of 256 pretrained models were trained for 20 epochs with Callback monitoring accuracy of validation set.



Hyperparameter Tuning

Training parameters were decided by using the trial-and-error approach. Various number of layers, size of filters and kernels, learning rate, minibatch size, dropout rate, number of epochs were experimented by monitoring models performance. Also, to prevent overfitting concept of Early Callback was implemented. Value of parameters mentioned above minimized the gap between training and validation accuracy.
